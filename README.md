<h3>Общее описание</h3>
Спасибо за интересную задачу, было интересно делать. Реализовали весь необходимый функционал. Запуская приложение, можно пройти по всему флоу сбора данных. Первое, что нужно - развернуть на компьютере Chrome браузер в режиме разработчика. Для этого нужно запустить в консоли <code>"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="C:\temp\chrome_debug_profile_9222"</code>. Если вы планируете запускать несколько параллельных процессов (а вы хотите!), открывайте нужно кол-во окон браузера с разными портами и user_data. В main файле можно отредактировать поисковые запросы, на которых будут работать процессы.

В проекте мы выступаем в качестве компании-подрядчика, которая делает для московской риелторской компании модель для предсказания предварительной стоимости квартир для новых клиентов, а также помогает с общим анализом факторов повышения/понижения стоимости недвижимости. Для этого мы парсим сайт платформы для продажи квартир Домклик при помощи Selenium, для ускорения этого процесса мы использовали библиотеку multiprocessing, которая позволяет развернуть несколько кластеров selenium собирающих информацию параллельно. Домклик имеет крайне хорошую анти-бот систему, для ее обхода мы использовали Selenium Stealth, а также ряд заголовков и подмену кэша. При отладке пректа мы использовали Selenium Wire - это внешний слой, который перехватывает уходящие запросы и позволяет их логировать и анализировать.

После сбора данных (>10к уникальных квартир) мы приступили к дообогащению нашего датасета при помощи сторонних API, используя 5 уникальных API методов

Также в рамках выполнения проекта были использованы регулярные выражения, логирование, которое можно настривать через конфиг и, конечно, selenium для эмитации работы пользователя (по-другому домклик не парсится)
